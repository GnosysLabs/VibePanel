# VibePanel

A premium, high-performance, and secure server management dashboard designed for modern hosting environments. Monitor native PM2 scripts, inspect active Docker stacks, synchronize reverse proxy configs, and automate sysadmin remediations using an AI-guided assistant—all styled in a monochrome, high-contrast dark zinc aesthetic.

---

## Features

- ⚙️ **Real-Time Telemetry**: Live tracking of host CPU utilization, memory allocations, network I/O streams, and NVMe disk quotas.
- 🐳 **Docker Daemon Manager**: Inspect volume bindings, read image metadata, and trigger start/stop/restart commands natively on the host docker daemon.
- ⚡ **PM2 Native scripts**: Monitor cluster instances, read process runtimes, reload scripts, and stream stdout/stderr log output files directly from disk.
- 🌐 **Reverse Proxy Configuration**: Track active rules and generate fully formatted, production-ready Nginx server blocks or Caddyfile configurations side-by-side.
- 🧠 **AI Sysadmin Assistant**: A floating, biometric-style agent bubble equipped with live-telemetry prompt injections. Ask diagnostic questions or execute 1-click server fixes generated by the LLM (powered by Anthropic, OpenAI, or OpenRouter).
- 🔒 **Edge-Guard Security**: Secure Session HTTP-Only cookie verification checked at the routing layer via Next.js Middleware.

## 🤖 Agent Installation (First-Class Workflow)

If you are using an autonomous AI coding agent (such as **Claude Code, Cursor, Windsurf, or Co-Pilot**) to set up and deploy applications on your server, copy and paste the prompt below to your agent. It will handle the entire environment check, configuration setup, and persistent daemon registration automatically.

### Copy-Paste Prompt for your AI Agent:

```text
Help me deploy and configure VibePanel (a Next.js server dashboard) on this host. 

Please perform the following operations:
1. Clone the Codebase: Clone the VibePanel repository from https://github.com/GnosysLabs/VibePanel.git into a deployment directory.
2. Environment Check: Verify if node (v18+), npm, docker, and pm2 are installed on this server. Report the versions.
3. Setup Credentials: In the cloned root directory, create a secure `.env.local` file. Generate a strong random password and define the credentials:
   ADMIN_USER=admin
   ADMIN_PASSWORD=<generate_strong_random_password>
   Keep a record of the generated password for my reference.
4. Install & Build: Run `npm install` followed by `npm run build` to compile the optimized production bundle.
5. Persistent Registration: Set up PM2 to manage VibePanel persistently in the background. Start it using:
   `pm2 start npm --name "vibepanel" -- run start -- -p 3000`
   Ensure the PM2 configuration is saved so it survives system reboots:
   `pm2 save`
6. Reverse Proxy Suggestion: Tell me what ports are open and provide a recommended Nginx server configuration or Caddy block to reverse-proxy port 3000 to a public domain with SSL.
```

## 🛠️ Manual Installation

If you prefer to configure VibePanel manually, follow these commands:

### 1. Clone & Set Up Environment
```bash
git clone git@github.com:GnosysLabs/VibePanel.git
cd VibePanel
```

Create a `.env.local` file in the root directory:
```env
# Define secure dashboard credentials
ADMIN_USER=admin
ADMIN_PASSWORD=your_super_secure_password
```

### 2. Install Dependencies & Build
```bash
npm install
npm run build
```

### 3. Run with PM2
To keep VibePanel running persistently in the background:
```bash
pm2 start npm --name "vibepanel" -- run start -- -p 3000
pm2 save
```

## 🛡️ Security Configuration

VibePanel is private by default. 
- All pages and APIs are protected at the router level by Next.js Edge Middleware.
- Authentic sessions are signed using the WebCrypto API (standard HMAC-SHA256 JWT tokens) and stored in secure, browser-inaccessible `HttpOnly` cookies.
- If default credentials (`admin`/`vibepanel`) are left active, a critical security warning will highlight inside the dashboard console stream.
